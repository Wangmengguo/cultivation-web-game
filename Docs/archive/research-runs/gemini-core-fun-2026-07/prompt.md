# Gemini Deep Research Prompt：增量修仙游戏为何有趣、解决什么问题

你是一名兼具游戏设计研究、玩家心理、增量游戏系统分析与中国修仙题材研究能力的高级研究员。请进行一次多来源、证据驱动的 Deep Research，回答本项目的第一个核心问题：

> **一个以数值增长、资源生产和自动化为主要表现的“增量修仙／宗门经营游戏”，怎样才能持续有趣？它相对于普通增量游戏、普通修仙游戏和传统角色扮演游戏，具体解决了哪些玩家体验与设计表达问题？**

报告必须直接回答这个问题，不要把主体写成竞品罗列、修仙文化介绍、商业分析或完整游戏策划案。

## 一、项目语境

这是一个 Web 宗门增量经营游戏的早期设计研究。暂定核心体验是：玩家以宗主身份建立并反复重构一套会自行运转的宗门经济，通过有限人口的岗位调度、跨资源生产与加工、投资、规则里程碑、自动化，以及带公开期限的任务／事件，应对不断升级的外部压力。

暂定核心循环：

1. 读取库存、净产速、容量、任务目标与期限；
2. 判断瓶颈和未来缺口；
3. 调整人口岗位、加工配方、投资顺序和库存安全线；
4. 让系统持续生产、消耗、建设和研究；
5. 通过里程碑获得倍率、新转化方式或自动化；
6. 完成任务或承担失败后果；
7. 让任务结果回到资源网络，形成下一轮条件。

这些都只是**待验证假设**，不是预设结论。请特别检验：

- 有限人口重配、资源链、几何成本、里程碑和自动化，是否真的能形成持续决策，而不只是最优解计算；
- 带期限且事前可读的任务／事件，是否能打破纯增量游戏的固定最优序列，在长期复利与近期交付之间制造有意义的取舍；
- 数值、卡片、趋势、日志和系统状态，能否承载“我是宗主、宗门在成长”的角色认同与修仙叙事；
- 不同修炼／宗门阶段如何通过**规则和决策空间的变化**体现成长，而不是只把数字整体放大；
- 自动化能否接管已经掌握的旧问题、释放注意力，同时把玩家推向新的高层决策；
- 上述组合是否会因信息过载、重复劳动、等待、假选择、随机惩罚、数值失控或叙事割裂而失败。

## 二、必须回答的研究问题

### A. 先给出核心答案

请用一段不超过 300 字的中文结论，明确回答：

1. 这种游戏的趣味性究竟来自什么；
2. 玩家反复进行的、需要思考的“问题解决活动”是什么；
3. 它具体解决了普通增量游戏与修仙题材表达中的哪些问题；
4. 在什么条件下，这个命题不成立。

不要用“丰富玩法”“增强沉浸感”“提高可玩性”等空泛措辞；必须指出因果机制。

### B. 识别真正的问题

分别分析以下三类问题，并区分“已有证据”“合理推断”和“本项目仍需验证的假设”：

1. **普通增量游戏的体验问题**：低能动性、固定最优路线、等待替代决策、重复点击、指数膨胀导致早期选择失去意义、自动化后无事可做、离线进度惩罚或焦虑、信息不透明、内容消耗后疲劳等。
2. **修仙题材的表达问题**：漫长修炼时间、境界跃迁、资源积累、洞府／宗门建设、弟子分工、机缘与劫难、因果延续，如何转成玩家可理解和可干预的系统；哪些修仙体验无法只靠数值表达。
3. **二者结合的新风险**：把修仙变成换皮表格、任务沦为资源缴纳清单、随机事件否定长期规划、系统很多但没有真正选择、剧情与经济系统互不影响、阶段扩展造成认知负担等。

### C. 解释“趣味性是如何发生的”

不要只列功能。请建立一条可检验的因果链：

```text
玩家可读的状态／压力
→ 玩家形成预测和计划
→ 做出有代价的选择
→ 系统产生及时、可解释的反馈
→ 瓶颈或规则发生变化
→ 玩家更新心智模型并获得掌握感
→ 旧问题被自动化，新问题出现
```

请说明每一环需要哪些设计条件、常见失败方式，以及如何观察玩家是否真的感到有趣。重点分析：有意义的选择、控制感、胜任感、好奇心、期待、惊喜、风险管理、长期规划和身份认同之间的关系。不要把留存、在线时长或付费等商业指标直接等同于趣味性。

### D. 回答“解决了哪些问题”

制作一张“问题—机制—心理作用—证据—副作用—验证方法”矩阵。至少覆盖：

- 固定路线与低自由度；
- 等待和重复劳动；
- 指数增长缺少意义；
- 自动化后失去参与感；
- 修仙阶段只剩数值门槛；
- 宗主身份不成立；
- 叙事与数值系统割裂；
- 任务系统随机性过高、玩家无法规划；
- 中后期信息过载；
- 失败不可理解或不可挽回。

每个“解决方案”都要说明它是否真正解决问题，还是只把问题转移到另一层。

### E. 用案例验证，而不是照抄案例

请主动搜索并分析具有代表性的作品，优先选择能够观察到机制、玩家行为或开发者论述的案例。候选范围可包括但不限于：Kittens Game、Evolve Idle、Melvor Idle、Antimatter Dimensions、Universal Paperclips、A Dark Room、Realm Grinder、NGU Idle、Increlution、Idle Loops、Magic Research／Magic Research 2、Orb of Creation，以及与宗门经营、修仙成长或长期角色养成有关的作品。

对每个采用的案例只回答：

1. 它让玩家持续解决什么问题；
2. 哪个机制改变了玩家的决策或心智模型；
3. 它如何通过阶段、里程碑或自动化更新玩法；
4. 它暴露了什么失败模式；
5. 哪些结论可迁移到本项目，哪些不可迁移。

请至少包含：

- 3 个成熟增量游戏的正向案例；
- 2 个能说明失败模式或玩家争议的反例／混合案例；
- 2 个修仙／宗门题材案例，用来检验题材承诺和身份感，而不是只比较美术与世界观。

## 三、证据与研究方法

1. 优先使用一手资料：官方游戏、官方 Wiki／手册、开发者文章或访谈、演讲、补丁记录、公开源码与设计说明。
2. 玩家评论、论坛、Reddit、Steam 评论等只能作为体验信号；应说明样本偏差，不得把个别意见写成普遍事实。
3. 可使用游戏设计与心理学研究，但要避免把相关性包装成因果。优先引用同行评审论文、GDC 演讲、学术著作或可信机构资料。
4. 对每项关键结论给出可点击来源，标注发布日期或访问日期；尽可能交叉验证。不要引用无法核查的二手聚合文、SEO 内容农场或 AI 生成摘要。
5. 明确区分：来源直接支持的事实、跨案例归纳、你基于证据作出的推论、本项目需要原型测试的假设。
6. 不要因为某机制在知名作品中存在，就断言它一定有趣；要解释适用条件和反例。
7. 研究主要面向单机／轻联网、非强付费驱动的 Web 或 PC 游戏；若引用手游免费增值设计，必须剥离商业化压力带来的干扰。

## 四、期望交付结构

请用中文完成，术语首次出现时附英文。按以下顺序输出：

1. **核心答案（≤300 字）**
2. **研究边界与“趣味性”的操作性定义**
3. **玩家究竟在反复解决什么问题**
4. **普通增量游戏、修仙题材及二者结合的痛点诊断**
5. **趣味性因果模型**
6. **问题—机制—心理作用—证据—副作用—验证方法矩阵**
7. **案例比较与反例**
8. **对本项目暂定核心循环的逐项审查**：保留、修改、删除或仍待验证，并说明理由
9. **阶段设计原则**：开局、中期、高阶阶段分别应该引入什么新决策，旧问题如何被自动化，如何控制信息量
10. **叙事与身份原则**：如何让数值变化成为宗门故事与宗主选择的结果；同时指出数值无法承担、必须由文本／事件／角色关系补足的内容
11. **失败模式与红线**
12. **最小验证方案**：提出 3–5 个低成本纸面／表格／可点击原型实验。每个实验包含假设、变量、任务、观察指标、失败判据和下一步决策
13. **最终判断**：用“成立／有条件成立／不成立”评价“任务与事件驱动的宗门增量经营能够成为持续有趣的核心玩法”，并给出最关键的三条依据与最大不确定性
14. **参考资料**

## 五、质量要求

- 报告要帮助设计团队做取舍，而不是堆积灵感。
- 强迫自己回答“为什么”“通过什么机制”“代价是什么”“如何证伪”。
- 若资料不足，请直接写“证据不足”，不要用常识补成确定结论。
- 不要输出具体数值平衡表、完整世界观、商业化方案或技术选型，除非它们是解释核心问题不可缺少的证据。
- 结论应能直接转化为玩法原则、原型假设和否决标准。
