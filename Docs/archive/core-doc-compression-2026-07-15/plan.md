# 当前计划

状态：Current

## 目标

建立一套完整、可计算、可纸面验证的任务驱动增量游戏框架，再通过开局与高潮两个端点确认它是否有趣。

## 执行顺序

### 1. 完成项目重置

- 归档旧文档和原型。
- 重建当前文档入口。
- 清理当前研究中的旧项目内容。
- 记录新的权威顺序和决策编号。

### 2. 完成完整系统框架

- 定义十个模块及接口。
- 确定资源更新顺序与动作结算。
- 确定倍率组合、几何成本和里程碑语义。
- 确定任务期限如何进入投资和产能计算。
- 确定离线与在线共用的结算语义。

### 3. 设计开局纸面段

- 从第一次手动操作开始。
- 依次引入自动设施、人口、岗位和加工。
- 验证早期至少两次瓶颈迁移。
- 记录产速、成本、回本时间和操作理由。

### 4. 设计高潮纸面段

- 使用同一套系统原语。
- 同时呈现多资源、多岗位、自动化和任务压力。
- 验证至少两条可行但后果不同的策略。
- 让成功和失败都能由生产历史解释。

### 5. 连接完整过程

- 设计中段解锁顺序。
- 安排任务和事件节奏。
- 确保资源用途持续存在。
- 让旧层逐步自动化，新层持续产生新瓶颈。

### 6. 构建全程数值模型

- 用纯模拟核心运行固定策略。
- 校验资源守恒和离线一致性。
- 导出产速、库存、停工、回本和任务压力。
- 迭代到全程无死锁、无无界漏洞且路线有差异。

### 7. 选择并实现 Web 底座

- 根据完整领域模型评估技术栈。
- 保持模拟核心与界面分离。
- 一次实现开局、中段和高潮的完整流程。
- 在实现后进行真人试玩和数值回放对照。

## 当前停止线

在完整系统框架通过前，不进入具体内容设计和代码实现。在开局与高潮纸面段通过前，不开始拼接完整过程。
