# 阶段门

状态：Current

## 阶段 0：项目重置

目标：清除旧规格的当前效力，只保留增量机制研究。

通过条件：

- 旧文档和原型已整体归档为试验资料。
- 当前入口不再引用旧人物、旧成长规则和旧危机。
- 当前研究目录只保留增量算法与 Web 底座研究。
- 新决策台账明确项目已重置。

## 阶段 1：完整系统框架

目标：不写具体内容，先定义时间、人口、岗位、资源、加工、设施、成本、任务、事件、成长、自动化、离线和反馈之间的完整关系。

通过条件：

- 每个模块有输入、输出和对其他模块的依赖。
- 资源更新顺序、动作结算和倍率组合没有歧义。
- 任务期限能够进入产能与投资计算。
- 不需要借助具体人物或剧情才能说明循环为什么成立。

## 阶段 2：开局纸面段

目标：验证玩家如何从第一次手动产出进入人口岗位调度、第一条加工链和第一次稳定生产。

通过条件：

- 玩家在短时间内至少经历两次瓶颈迁移。
- 第一项自动化来自玩家已经手动解决过的问题。
- 产速、成本和预计完成时间始终可读。
- 不依靠文本奖励也愿意继续增长。

## 阶段 3：高潮纸面段

目标：验证多层资源、多个岗位、自动化和高压任务同时存在时，玩家仍能理解局面并作出不同取舍。

通过条件：

- 高潮不是单一总数阈值，也不是随机结果表。
- 任务紧迫性真实改变投资和库存策略。
- 至少两条策略可以完成目标，但成本和后续状态不同。
- 玩家能说明成功或失败由哪些生产决定造成。

## 阶段 4：完整过程拼接

目标：设计开局与高潮之间的中段，形成连续的资源层、任务节奏、里程碑和自动化升级。

通过条件：

- 每次解锁都有前因和新用途。
- 没有长期闲置资源。
- 旧层逐步自动化，新层持续制造新瓶颈。
- 中段不是重复开局，也不是只为拖长时间。

## 阶段 5：完整数值模型

目标：把全程规则做成可重复模拟的纯数值模型。

通过条件：

- 四类固定策略脚本均可完整运行。
- 无负库存、永久死锁、无界漏洞和无法解释的产出。
- 在线与离线按同一语义结算。
- 产速、成本、回本时间和任务压力能被导出审计。

## 阶段 6：Web 技术底座

目标：在完整规则已经稳定后选择长期可维护的实现底座。

通过条件：

- 技术栈不强迫领域模型服从威望树或固定 UI。
- 模拟核心独立于界面。
- 存档迁移、测试和离线计算有明确方案。
- 依赖可重复安装，许可证允许计划中的使用方式。

## 阶段 7：可玩初版

目标：实现完整流程，而不是只实现几个孤立系统。

通过条件：

- 开局、中段和高潮可以完整走通。
- 任务和事件真实驱动资源调度。
- 自动化减少重复操作但不替玩家决定新目标。
- 试玩能够区分数值问题、内容问题和界面问题。
