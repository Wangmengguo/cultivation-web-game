# Game Spec 输入清单

状态：Current

## 已确认

- 游戏是 Web 端宗门增量经营游戏。
- 数值生产、迭代和增长是核心表现，不以场景单位运动作为初版核心。
- 核心算法建立在人口岗位、跨资源加工、几何成本、规则里程碑和自动化之上。
- 任务和事件是相对纯增量游戏的主要差异。
- 初版先定义完整框架，不先实现几个孤立内容。
- 完整框架后先做开局和高潮两个纸面段，再连接中间过程。
- 重置前全部人物、成长、危机、概率和数值不继承。
- Kittens Game 只研究机制，不复制源码或内容。
- Profectus 0.7 已经独立复核，不作为长期底座，也不作为默认原型底座。

## 待验证假设

- 任务期限能够让人口调度和投资顺序产生比纯增量更强的决策张力。
- 只用数值、卡片和趋势反馈，也能让构筑生产能力本身有趣。
- 开局与高潮可以共享同一组数值原语，而不需要两套规则。
- 自动化可以降低重复操作，同时保留任务到来时的重新规划。
- 资源网络可以保持可读，而不会随着内容增加退化成表格负担。

## 当前开放问题

- 开局的第一种手动操作是什么。
- 第一种自动设施、第一批人口和第一条加工链如何依次解锁。
- 任务以交付、容量、生产结构、区域建设还是组合目标为主。
- 高潮需要多少资源层和并行任务才足以产生压力，又不失去可读性。
- 成本倍率、产速、容量、回本时间和任务期限的目标区间。
- 长期结构是否需要重置、传承或其他周目机制。
- 正式技术栈如何在完整数值模型稳定后选择。
