# 项目简报

状态：Current

## 产品定义

这是一个由任务和事件驱动的 Web 宗门增量经营游戏。

玩家不直接操控场景中的单位行动，而是管理人口岗位、生产速度、资源加工、投资顺序、能力里程碑和自动化规则。宗门持续运转，任务和事件不断提出有期限的要求，迫使玩家改变生产结构并检验已经构筑出的能力。

## 与纯增量游戏的区别

Kittens Game 的核心压力主要来自资源需求、人口消耗、容量、科技和更高层解锁。本项目保留这些增量骨架，但增加一条外部节奏轴：任务和事件会公开目标、期限与代价，使玩家不能永远只按最高长期收益发展。

因此本项目不是纯等待增长，也不是在增量游戏外再挂一套随机剧情。任务和事件必须进入资源网络，改变岗位分配、库存安全线、投资时机和自动化优先级。

## 五个承重系统

1. 人口岗位调度。
2. 跨资源生产与加工链。
3. 几何成本、规则里程碑与旧层自动化。
4. 有期限的任务和事件压力。
5. 可解释、可回放、可模拟的数值反馈。

## 初版原则

初版先搭完整玩法框架，不先实现若干孤立内容。完整框架至少要说明时间、人口、岗位、资源、加工、设施、成本、任务、事件、成长、自动化、离线进度、反馈和数值验证如何连接。

完成框架后，先在纸面上分别做最开始的环节和最高潮的环节。两端都能成立后，再设计中间过程并拼成完整体验。

## 当前不继承的内容

重置前的具体人物、成长方式、危机场景、战斗输入、概率表、资源名、时间单位和技术选型全部不继承。当前文档不得以旧内容作为例子继续推演。

Kittens Game 只作为机制研究对象。其源码不进入本项目代码库，也不作为派生开发底座。
