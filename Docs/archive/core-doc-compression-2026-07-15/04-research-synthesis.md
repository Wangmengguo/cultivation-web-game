# 增量玩法研究综合

状态：Current

## 结论

当前最可靠的玩法骨架不是一套新发明的概率模型，而是：

```text
有限人口重配
→ 基础资源与加工资源互相依赖
→ 几何成本迫使玩家选择投资时机
→ 规则里程碑改变生产方式
→ 自动化接管旧问题
→ 任务和事件迫使玩家重新安排增长路线
```

前三项和自动化已有成熟作品长期实现。最后一项是本项目的差异，也最需要独立验证。

## Kittens Game 初期实际做了什么

根据 Kittens Game 官方 Wiki，开局并不是立刻给玩家一张庞大资源树，而是逐步建立控制权：

1. 玩家最初只有手动采集基础资源的按钮。
2. 累积少量资源后建造第一种自动生产设施，第一次从点击进入持续产出。
3. 基础资源既是生存消耗，也是设施成本和第一种加工材料。
4. 第一批人口出现后，人口可以被分配到不同岗位，资源增长开始由岗位结构决定。
5. 科学和新建筑逐步解锁更多岗位、容量与加工，玩家从单资源转向多资源轮换。
6. 容量、季节和人口消耗制造安全线，阻止玩家只看总量上涨。

## 三组算法为什么重要

### 人口岗位和净产出

Kittens Game 与 Evolve Idle 都把人口视为有限生产单位。玩家不是逐次点击每个动作，而是在岗位之间调人，使不同资源的净速发生变化。

```text
岗位产速 = 单位产出之和 × 设施修正 × 全局修正
资源净速 = 总产出 - 总消耗
```

这个结构提供持续决策，因为给一个岗位加人必然意味着另一个岗位少人。

### 跨技能资源链和完整动作

Melvor Idle 证明了基础采集、加工和用途可以形成长期互相支撑。对消耗型加工，结算必须受完整动作和库存约束。

```text
可执行次数 = min(时间允许次数, 各输入允许次数)
```

这能防止负库存，也能让停工原因成为玩家可解决的瓶颈。

### 几何成本、里程碑和自动化

Antimatter Dimensions 等作品使用递增成本控制投资节奏，并用关键里程碑提供规则跃迁。

```text
下一件成本 = 基础成本 × 成本倍率^已拥有数量
```

连续的小升级提供平滑增长，关键等级提供新转化、保留规则或自动化。玩家获得的不是永远点击得更快，而是逐步不用再手动处理旧层。

## 本项目必须新增的任务压力

Kittens Game 的早期目标主要由下一个建筑、科技和资源需求自然产生。本项目要加入显式任务与事件，但不能把它们做成独立抽奖层。

任务需要参与以下计算：

- 距离期限还有多少时间。
- 按当前净速能否交付。
- 为完成任务需要临时转移多少人口。
- 当前投资能否在期限前回本。
- 保留库存会牺牲多少长期增长。
- 成功与失败如何改变后续资源网络。

## 当前研究不能证明什么

- 不能证明把三组机制组合起来就一定好玩。
- 不能证明任务压力一定优于纯增量节奏。
- 不能证明任何具体成本倍率和时间长度适合本项目。
- 不能证明必须使用超大数或长期重置。
- 不能证明任何现成 Web 增量框架适合长期维护。

这些问题要通过完整框架、开局纸面段、高潮纸面段和全程数值模型依次验证。
