# Web 增量游戏底座研究

状态：Current Research / Profectus 已复核并排除

日期：2026-07-15

## 当前决定

Profectus 0.7 不作为长期底座，也不作为默认原型底座。项目不会以短期省事为理由，让现成框架反向塑造完整领域模型。

Grok 已完成独立复核，完整项目内记录见 [profectus-0.7-grok-verification-2026-07.md](profectus-0.7-grok-verification-2026-07.md)。

## Profectus 0.7 复核结论

- Galaxy Git 服务在验证窗口内对站点根路径、仓库路径和 `git ls-remote` 稳定返回 HTTP 502。
- Node 22.23.1 与便携 Node 21.7.3 的干净安装都在同一 Git 依赖处失败，说明主因不是 Node 主版本。
- `package.json` 的 `engines.node` 确实限定为 `21.x`，GitHub 与 Forgejo CI 也固定 Node 21。
- Node 22 在安装阶段只触发 `EBADENGINE` 警告，但官方完整依赖图下的构建并未验证通过。
- 官方声明 GitHub 仓库只是镜像，主要协作面与 Galaxy 依赖位于同一不可达主机，形成明显单点风险。
- 即使绕过 Galaxy，框架的一等公民仍是 layer、tree、prestige、conversion 和 formula，不是人口岗位与有序多资源生产链。
- 离线追赶把较大的 `diff` 广播给各层，没有提供按资源依赖图、库存耗尽点和动作边界切片的生产链语义。
- 0.7 只有标签，没有 GitHub Release；0.7 后只有少量修复提交，最后一次代码活动为 2025-07-17。

结论：502 已确认，Node 21 声明确认，Node 22 的完整兼容性只能部分确认。拒绝 Profectus 作为长期底座有充分技术依据。即使服务恢复，也不重新进入默认候选。

可以独立借鉴的只有 Formula、Requirement、Modifier、存档迁移钩子、多存档槽，以及 `preUpdate`、`update`、`postUpdate` 的职责切分。不得照搬 Galaxy、默认威望树和粗粒度离线结算。

## 长期底座必须满足的条件

- 模拟核心独立于 UI。
- 适合 DOM 数值面板、岗位调度和任务信息，而不是强制威望树界面。
- 多资源消耗链的在线与离线结算一致。
- 存档、迁移、导入导出和确定性测试可维护。
- 依赖来自稳定来源，可在干净环境重复安装。
- 许可证允许计划中的商业使用。
- 即使上游停止维护，也能以合理成本接管或替换。

## 当前候选策略

在完整系统框架和全程数值模型稳定前，不锁定正式技术栈。后续优先比较现代 Web UI 框架加独立 TypeScript 模拟核心，以及经过许可和维护审查的增量工具库。

Phaser 等 Canvas 引擎只在动态场景成为实际需求时再评估，不负责当前数值真值。

## 研究纪律

框架能减少工程重复，但不能验证玩法。算法的成熟度、框架的维护性和本项目的乐趣必须分别举证。
