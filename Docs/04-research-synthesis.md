# 现有研究综合

状态：Evidence + Hypothesis

## 研究能回答什么

现有材料证明，多名角色分工、向具体角色投入资源、增量资源链、离线推进和转生加速都在成熟游戏中出现过。修仙竞品也表明“宗主、弟子、岗位、丹药、突破”等表达已被同类产品用于对外沟通。

这些证据没有证明上述机制组合起来一定好玩，也没有证明“世代经营、定期大考、人物一生与跨代底蕴”组合后能够产生足够强的策略与情绪回报。这个问题必须由纸面推演和试玩验证。

## 工作分配与组织管理

RimWorld、Dwarf Fortress 和 Oxygen Not Included 提供了清晰的工作分配范式：玩家决定哪些个体可以做什么、优先做什么，单位属性和工作匹配影响结果。Kittens Game 和 Evolve 则证明，可分配人口能够成为资源生产的根约束。

可以支持的判断：宗主给弟子安排岗位是成熟管理动作；弟子差异需要影响岗位效果；长期安排适合持续执行。

不能推出的结论：本项目必须使用优先级矩阵、复杂工作 AI、地图寻路或大量岗位；工作分配本身也不保证产生爽感。

主要原始材料：[宗门管理玩法详稿](./archive/originals/research/verified-play-patterns-for-sect-management.md)、[Idle 资源循环笔记](./archive/originals/research/idle-resource-loop-source-notes.md)。

## 向具体个体投入稀缺资源

Fire Emblem: Three Houses、Darkest Dungeon、XCOM 和 Football Manager 都包含面向具体角色的训练、装备、治疗或培养选择。资源有限时，玩家会形成偏爱、路线和机会成本。

可以支持的判断：丹药、指导机会或突破资格由玩家指定给某个弟子，具有成熟玩法依据；个体投资比自动平均增长更容易形成选择理由。

不能推出的结论：丹药必然是第一版最合适的资源；只能有一个培养对象；培养必须通过固定按钮或固定数量表达。

主要原始材料：[宗主玩法转译摘要](./archive/originals/research/validated-sect-leader-play-patterns.md)、[宗门管理玩法详稿](./archive/originals/research/verified-play-patterns-for-sect-management.md)。

## 增量资源链与阶段兑现

Progress Knight 证明了“旧阶段在下一轮明显变快”的增量反馈；Melvor Idle 证明技能和资源互相供给能够构成长线规划；Kittens Game 与 Evolve 展示了人口、加工和多层成长如何互锁。

可以支持的判断：采集、加工、培养之间形成短资源链是成熟结构；重大成长需要带来肉眼可见的结果；旧阶段变快可以作为候选爽点。

不能推出的结论：突破一定等同 prestige；必须重置修为；必须使用倍率总线、指数成本或两层转生；Progress Knight 必须成为唯一骨架。

主要原始材料：[GitHub 放置源码对标](./archive/originals/research/github-idle-source-benchmark.md)、[Idle 资源循环笔记](./archive/originals/research/idle-resource-loop-source-notes.md)。

## 低频决策与可观察反馈

开罗经营游戏、Football Manager 和多种管理游戏都把大量执行交给系统，玩家负责少量高影响决定，再通过项目结果、比赛报告、角色状态或场景变化读取后果。

可以支持的判断：宗主身份适合“下令后观察结果”的节奏；重要决定需要清楚说明后果；执行过程可以自动化。

不能推出的结论：日志必须成为界面中心；所有结果必须确定无随机；游戏必须使用固定的短、中、长三层时间参数。

主要原始材料：[宗门管理玩法详稿](./archive/originals/research/verified-play-patterns-for-sect-management.md)、[旧核心架构](./archive/originals/core-architecture.legacy.md)。

## 修仙题材与竞品边界

《最强祖师》《了不起的修仙模拟器》《修仙家族模拟器》以及多款修仙 idle 产品，已经采用宗主经营、弟子培养、生产岗位、丹药、境界和离线成长等表达。它们也覆盖了抽卡养成、深模拟、单角色放置和多人战力等不同路线。

可以支持的判断：修仙题材已经被用于承载组织经营和增量成长；宗主与弟子的语义有现成产品表达可参考；避免一开始进入抽卡战力、跨服和深模拟，有利于控制范围。

不能推出的结论：某个商业空位已经被证明；目标玩家会为当前候选玩法付费；竞品商店描述中的功能一定产生留存。

主要原始材料：[修仙竞品研究](./archive/originals/research/cultivation-idle-competitors.md)。

## 研究与当前 Game Spec 的关系

正式 [Game Spec v0](./GAME_SPEC.md) 已选择世代经营与定期大考作为核心。现有研究可以支持其中的工作分配、定向资源投入、长期自动执行、阶段性成长和结果反馈。它不能直接证明寿元压力、人物一生、跨代传承与大考组合后仍然好玩。

此前“宗主定向培养弟子”的候选命题已经被替代。定向培养仍是世代经营中的一类手段，需要服务宗门长期积累和危机应对，不能再单独约束整个游戏方向。

## 仍然缺少的证据

- 目标玩家是谁，他们目前玩哪些游戏，以及他们期待多长的单次体验。
- 对参考游戏前期真实操作和反馈节奏的结构化试玩记录。
- 寿元、人物经历、跨代底蕴和大考组合起来是否仍然清晰、有取舍。
- 弟子需要多少个体性，才能让资源分配产生情感，而不只是算最优值。
- 日志、场景变化、角色反馈和数值变化中，哪一种最能承担兑现感。
- 当前竞品的用户评价、留存代理指标、价格和商业模式证据。

这些缺口分别属于 Game Spec 研究、纸面验证和产品定位，不应通过继续扩写架构文档来填补。
