## 当前阶段

游戏核心方向已确认为世代经营与定期大考：弟子寿数有限，宗门底蕴跨代保留，公开或可预期的大事件检验此前经营。[Game Spec v0](./GAME_SPEC.md) 已于 2026-07-11 通过阶段 1 验收。Spec 已吸收宗主权力与弟子自主性、战斗战略控制、多种经营哲学和第一段完整验证体验七阶段等修订；项目简述吸收了以一天为基础的时间循环与 5 到 10 分钟节奏假设、性格属性状态三类人物信息、全力攻击攻守兼备全力防守三姿态与无撤退阈值、规则与动态与 AI 的边界，以及第一段完整体验的 Make it work 内容原则。2026-07-11 又统一了战斗输入口径与临界撤退判定口径并正式通过阶段 1。2026-07-11 还确认了在世积累—寿元限制—死亡终止未来—已有成果交接关系：弟子活着时持续为宗门沉淀成果，寿元必须真实改变玩家对人、时间、机会与资源的分配而不是只是定期退场计时器，死亡终止未来贡献但凭空不生成传承，死亡时发生已有成果的交接、保留、丢失或中断。第一轮纸面验证主问题据此改为有限寿元是否真实改变玩家对人、机会与资源的分配，以及在世积累与死亡损失能否共同形成可归因跨代因果。第一轮中场景三“沈砺的最后一炉”已完成纸面裁决，确认有限寿元真实改变系统中的选择（人物、时间、资源、机会与宗门风险分配均被改变），人物原因与宗门原因无需互相剥离，原型概率与数值不升级为最终平衡；场景二“沉锋谷最后的引路人”已完成第一轮纸面结构收口，正式采用十五个月 v2 结构（岳徊剩余 3 个月、唯一延寿丹增加 12 个月、苏砚前 6 个月补齐硬资格、之后才通过引路图研习或本人带队提高熟练度），原六个月 v1 结构已废止——场景二作为后续明确例外覆盖旧的“三个场景统一六个月”初稿。沉锋谷每月资源刷新上限固定为 2 份，不随队伍人数增加；合格成员增加的是战力与伤势安全，每名参队者必须各自满足硬资格，强行派入不合格者会在公开规则下直接重伤濒死、中止历练且不取得资源；六名独立偏好模拟得到使用延寿丹5、保存1，两类选择均可辩护。场景二至此停止继续微调数值，保留真人测试入口，主流程进入三场景联合宗门局面（归元真人、岳徊、苏砚与沈砺共享一枚延寿丹、庚金髓、培养资源、可参战月份与同一次大考压力），联合局面的具体时间与人物数值尚未确认，不在同步阶段自行创造。阶段 1 已完成，不再等待复核。

阶段 1 已通过，下一阶段是纸面玩法验证。在此之前不写代码，不选技术栈，不写详细数值，也不展开炼丹、炼宝、外交、地图、神兽、比武和门派挑战等完整系统。正式规格以 GAME_SPEC.md 为准，本 README 不复述其内容。

## 文档效力

文档出现冲突时，按下面的顺序处理：

1. 用户在最新对话中的明确决定。
2. [决策台账](./06-decision-register.md)中的已确认事项。
3. GAME_SPEC.md 中与决策台账一致的当前规格。
4. 本目录中的项目简述、阶段门、研究综合和 Game Spec 输入。
5. `archive/originals/` 中的历史原文。

归档材料只用于追溯。它们可能包含有价值的研究，也可能包含已经被否决的玩法、过早确定的参数或失效的技术安排。引用归档内容时，必须先在决策台账中确认其当前状态。模型意见不能写成证据，只能作为待验证假设或开放问题处理。

## 文件状态

本资料库统一使用以下状态：

| 状态 | 含义 |
| --- | --- |
| 已确认 | 用户明确确认，当前可以作为工作前提 |
| 证据 | 有来源支撑的外部事实或成熟游戏机制 |
| 待验证假设 | 可以进入 Game Spec 或测试，但尚未证明适合本项目 |
| 开放问题 | 尚未决定，会影响后续方向 |
| 已否决 | 用户明确否定，不得重新作为默认方案 |
| 已被替代 | 曾经有效，已被较新的决定覆盖 |
| 归档 | 仅供追溯，不具备决策效力 |
