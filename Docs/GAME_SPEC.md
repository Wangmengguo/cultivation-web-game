# Game Spec v0.1

状态：Current / 重置后初稿

## 一句话核心

> 玩家建立并不断重构一套会自行运转的宗门经济，用人口调度、资源链、投资、里程碑和自动化完成越来越紧迫的任务与事件。

这是一个以数值生产、迭代和增长为主要表现的 Web 宗门增量经营游戏。玩家不直接操控场景单位行动；任务和事件也不是外挂的随机剧情，而是一条外部节奏轴，必须改变岗位分配、库存安全线、投资时机和自动化优先级。

## 设计边界

- 五个承重系统是人口岗位、跨资源生产与加工、几何成本与规则里程碑、有期限的任务事件，以及可解释、可回放的数值反馈。
- 初版先搭完整玩法框架，再分别纸面验证开局和高潮；两端成立后才连接中段。
- 重置前的人物、成长、危机、战斗输入、概率表、资源名、时间单位和技术选型全部不继承。
- Kittens Game 只作为机制研究对象；Profectus 0.7 不作为长期底座或默认原型底座。

## 核心循环

```text
读取库存、净产速、容量和任务期限
→ 识别当前瓶颈与未来缺口
→ 调整人口岗位、加工配方、投资和库存安全线
→ 生产、消耗、建设与研究持续结算
→ 到达规则里程碑，获得倍率、新转化或自动化
→ 完成任务或承担失败后果
→ 任务结果进入资源网络，形成下一轮条件
```

## 完整玩法框架

初版必须同时定义以下十个模块的接口。模块可以先用纸面和表格表达，但不能缺席。

### 1. 时间

- 生产以连续时间或固定 tick 推进。
- 任务和事件使用公开期限。
- 所有长期动作都能换算为预计完成时间。
- 在线与离线遵循同一组资源守恒和动作结算规则。

### 2. 人口与岗位

- 人口是有限、可重配的离散生产单位。
- 每个单位同一时间只承担一个主要岗位。
- 岗位调整必须立即改变产速、消耗或完成时间。
- 自动化只能替代已经掌握的重复调度，不能自动决定新任务的战略。

### 3. 资源

- 资源分为基础产物、加工产物、容量或条件资源。
- 每种核心资源至少有两个竞争用途。
- 每种持续产出都必须有消耗端、容量或解锁用途。
- 界面同时显示库存、净速、来源、消耗、预计耗尽和预计填满。

### 4. 生产与加工

- 基础岗位产生连续资源流。
- 加工按完整动作结算，输入不足时停止，不允许负库存。
- 生产链必须能形成瓶颈迁移，不能永远只缺同一种资源。
- 加工效率、动作时间和批量产出分别建模，避免全部压成一个倍率。

### 5. 设施与成本

- 常规重复设施采用几何成本。
- 小等级负责平滑增长，关键等级负责改变规则。
- 容量、产速、加工、任务准备和自动化之间必须争夺同一批资源。
- 买一项设施的价值需要用回本时间和任务期限共同评价。

### 6. 任务与事件

每个任务或事件都包含：出现条件、公开目标、期限、所需能力或资源、成功结果、失败结果、是否可重试，以及结果如何回到资源网络。

任务和事件不能只在末端抽取结果。它们必须提前改变生产计划，并让玩家在长期复利与近期交付之间取舍。

### 7. 成长与解锁

- 新层通过资源、设施、任务成果或规则里程碑解锁。
- 解锁应增加新的生产关系或新的控制权，不只是增加数值。
- 旧层在玩家证明掌握后逐渐自动化。
- 新层必须制造新的瓶颈，而不是只把旧数字整体放大。

### 8. 自动化

- 自动化从库存阈值、加工开关、优先级和批量购买开始。
- 自动化规则由玩家配置，系统负责执行。
- 任务和事件可以临时覆盖常规优先级，但不能替玩家自动选择目标。
- 自动化的主要奖励是释放注意力和人口，让玩家进入更高层决策。

### 9. 离线进度

- 离线计算按完整动作和资源约束结算。
- 当库存耗尽、容量填满、设施完成或规则触发时切分时间段。
- 回归汇总必须解释产出、消耗、停工和任务状态变化。
- 离线系统不得凭空替玩家进行未配置的投资或任务选择。

### 10. 反馈与验证

- 每个产速都能展开为基础值、岗位贡献、设施、里程碑和事件修正。
- 每次重要调整都记录调整前后的短期变化。
- 数值模型支持固定策略脚本和完整回放。
- 稳定性以资源守恒、无死锁、无无界漏洞和路线差异衡量，不以所有路线结果相同衡量。

## 算法基础

当前只采用三组经过公开实现或市场长期检验的结构：

1. Kittens Game 与 Evolve Idle 的人口岗位调度和净产出。
2. Melvor Idle 的跨技能资源链和完整动作结算。
3. Antimatter Dimensions 的几何成本、规则里程碑和旧层自动化。

具体公式、初始参数和任务强度必须重新建模。研究对象的源码、数值表和内容不能直接复制为本项目实现。

## 任务驱动的差异

纯增量模型主要问下一项投资如何提高长期增长。本项目还要问：在期限到来前，哪些长期投资必须推迟，哪些库存必须保留，哪些岗位必须临时调整，哪些自动规则需要改变。

任务压力必须满足三条：

1. 事前可读，玩家能估算缺口。
2. 过程可干预，调度和投资会改变结局。
3. 事后回流，结果成为新资源、新约束或新生产关系。

## 待验证假设

- 任务期限能够让人口调度和投资顺序产生比纯增量更强的决策张力。
- 仅依靠数值、卡片和趋势反馈，也能让构筑生产能力本身有趣。
- 开局与高潮可以共享同一组数值原语，而不需要两套规则。
- 自动化可以降低重复操作，同时保留任务到来时的重新规划。
- 资源网络在扩展后仍然可读，不会退化成表格负担。

## 当前未定义

- 具体人口形态与个体差异。
- 具体资源、设施、科技、任务和事件内容。
- 第一次手动操作、自动设施、人口和加工链的确切解锁顺序。
- 任务以交付、容量、生产结构、区域建设还是组合目标为主。
- 开局与高潮的时长。
- 高潮的资源层数、并行任务数和可读性边界。
- 成本倍率、产速、容量、回本时间、期限区间与是否需要超大数。
- 长期重置、周目或其他跨局结构。
- 美术和动态表现。
- 正式 Web 技术栈。

这些内容不是沿用旧方案，而是等待新框架下重新决定。
